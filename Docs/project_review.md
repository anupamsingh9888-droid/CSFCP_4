# Development Process

## 1. Planning
The development began by outlining the core mechanics of the game and identifying:
- The type of game (e.g., arcade, text adventure, puzzle).
- The required functions and components.
- How the game loop and input system should behave.

A rough flowchart or pseudocode was drafted before coding.

## 2. Setting Up the Structure
A basic project scaffold was created with:
- `game.py` containing the main logic.
- A loop structure to run the game.
- Functions or classes to organize code (e.g., Player, Enemy, GameState).

## 3. Implementing Core Mechanics
Key steps included:
- Designing the game loop:
  - Update → Render → Input → Repeat
- Handling user input (keyboard commands or button presses).
- Implementing game rules (movement, scoring, collision, etc.).
- Debugging incremental pieces as they were added.

## 4. Testing
During development:
- Each part of the game was tested individually.
- Edge cases were checked (invalid input, boundary positions, unexpected behavior).
- Adjustments were made to improve stability and user experience.

## 5. Finalization
After the main features worked consistently:
- Code was cleaned and commented.
- Functions were refactored for readability.
- Final gameplay tweaks and balancing were applied.
